## 1. 系统要求

| 项目 | 要求 |
|------|------|
| Android 版本 | **≥ 10**（API 29）|
| 运行设备 | **真实 Android 手机**（模拟器 AVD 不支持，见下方说明）|
| 权限 | `ACCESS_FINE_LOCATION`（连接指定 SSID 必须）|
| 屏幕方向 | **横屏**（强制 Landscape，不可旋转）|
| ESP32-S3 固件 | `TB6612motor_ESP32S3_WiFi.ino` 已烧录并运行 |

### 1.1 设备兼容性说明

#### 为什么必须使用真实设备（不能用 AVD 模拟器）

App 通过 `WifiNetworkSpecifier` API 连接 ESP32-S3 AP 热点，该 API 要求设备具备**物理 Wi-Fi 芯片**，并向系统申请 `TRANSPORT_WIFI` 类型的网络。  
AVD 模拟器没有真实 Wi-Fi 适配器，其网络是通过虚拟以太网桥接到宿主机的，`TRANSPORT_WIFI` 永远无法满足，连接会立即触发 `onUnavailable()` 回调并报错「找不到 Robot-ESP32 热点」。

| 运行环境 | 能否连接 ESP32 AP | 原因 |
|---------|-----------------|------|
| 真实 Android 手机 | ✅ 可以 | 有 Wi-Fi 芯片，`WifiNetworkSpecifier` 正常工作 |
| AVD 模拟器 | ❌ 不行 | 无 Wi-Fi 芯片，`TRANSPORT_WIFI` 不可用 |

#### 小米（MIUI）设备特别说明

以小米 Mix 4 为代表的 MIUI 手机完全兼容本 App，但有以下三点需要注意：

**① 「Wi-Fi 无法上网」弹窗**  
ESP32 AP 没有互联网出口，MIUI 会弹出提示：
> "Robot-ESP32 无法访问互联网，是否继续使用？"

选择 **「继续使用」** 即可。App 已调用 `bindProcessToNetwork()` 将流量强制绑定到 ESP32 AP，不受影响。

**② MIUI 后台省电策略**  
MIUI 的「神隐模式」可能在 App 切后台时杀掉进程，导致机器人失控。建议提前设置：
- 设置 → 应用设置 → 应用管理 → RobotController → **电池** → 选择「无限制」
- 在多任务界面下拉 App 卡片，锁定到后台（防止被清理）

**③ 位置权限路径**  
若首次弹窗时误选「拒绝」，可通过以下路径重新授予：
> 设置 → 隐私保护 → 权限管理 → 位置信息 → RobotController → 改为「使用期间允许」

## 2. 首次安装与权限授予

1. 安装 APK（Debug 或 Release 均可）。
2. **首次点击「连接」按钮时**，系统会弹出位置权限请求弹窗：

   ```
   允许"RobotController"在使用期间访问您的位置信息？
   ```

   必须选择 **「仅在使用该应用期间允许」**（或「始终允许」）。

   > **为什么需要位置权限？**  
   > Android 10+ 规定：应用通过 `WifiNetworkSpecifier` API 连接到指定 SSID（Robot-ESP32）时，系统强制要求 `ACCESS_FINE_LOCATION` 权限，否则系统会直接拒绝网络请求。此权限**仅用于 Wi-Fi 网络连接**，App 不会采集或上报任何位置数据。

3. 权限授予后，App 会**自动继续执行连接**，无需再次点击。

## 3. 启动与初始化连接

### 3.1 准备工作

```
① 给机器人上电 → ESP32-S3 自动启动 Wi-Fi AP（热点）
② 热点信息：
   SSID     : Robot-ESP32
   密码     : 12345678
   IP 地址  : 192.168.4.1
   WS 端口  : 81
```

> 等待 ESP32 LED 常亮（或串口打印 `WebSocket server started`），表示热点已就绪。

### 3.2 在 App 内连接

1. 打开 App，界面**自动横屏**，状态指示灯显示灰色「未连接」。
2. 点击中央面板的 **「连接」** 按钮（绑定图标 🔗）。
3. App 执行以下步骤（自动完成，无需手动操作）：

   ```
   Step 1：向系统申请绑定 Wi-Fi 网络（WifiNetworkSpecifier）
   Step 2：系统弹出选择/确认对话框 → 自动选择 Robot-ESP32
   Step 3：Wi-Fi 绑定成功后，App 将进程路由绑定到该网络
   Step 4：建立 WebSocket 连接到 ws://192.168.4.1:81
   Step 5：ESP32 发回握手确认 JSON
   ```

4. 连接成功后：
   - 状态指示灯变为 **绿色**
   - 显示文字 `已连接  192.168.4.1`
   - 「连接」按钮变为红色「断开」按钮
   - 双摇杆**解锁**，可操控电机

### 3.3 连接状态说明

| 状态指示灯颜色 | 含义 |
|---|---|
| 🔴 红色 | 连接错误，查看错误提示（如热点未找到）|
| 🟡 黄色 | 正在连接中… |
| 🟢 绿色 | 已连接，显示 ESP32 IP 地址 |
| ⚫ 灰色 | 未连接 |

## 5. 操控两个电机（双摇杆）

### 5.1 摇杆说明

| 摇杆位置 | 对应电机 | 控制逻辑 |
|---------|---------|---------|
| **左摇杆** | Motor A（左电机） | Y 轴上推 → 前进；下拉 → 后退 |
| **右摇杆** | Motor B（右电机） | Y 轴上推 → 前进；下拉 → 后退 |

> **注意**：摇杆 X 轴（左右）在当前版本**不使用**，两轮差速完全由双摇杆各自的 Y 轴独立决定（坦克式操控）。

### 5.2 坦克式差速驾驶

坦克式操控的核心原理：**两轮速度差决定转向**，左轮慢（或倒转）右轮快则向左转，反之向右转。左、右两轮互相独立，转弯半径由速度差的大小决定。

#### 基本运动指令

```
直行前进   → 左摇杆 ↑ + 右摇杆 ↑（两杆同时上推相同幅度）
直行后退   → 左摇杆 ↓ + 右摇杆 ↓（两杆同时下拉相同幅度）
原地左转   → 左摇杆 ↓ + 右摇杆 ↑（左轮后退 + 右轮前进）
原地右转   → 左摇杆 ↑ + 右摇杆 ↓（左轮前进 + 右轮后退）
缓慢左转   → 左摇杆 ↑ 幅度 < 右摇杆 ↑ 幅度（左慢右快，弧线左偏）
缓慢右转   → 左摇杆 ↑ 幅度 > 右摇杆 ↑ 幅度（左快右慢，弧线右偏）
```

#### 四个方向转弯详解

以下以机器人**从上方俯视**为视角，`L` = 左轮（Motor A），`R` = 右轮（Motor B）：

---

**① 向左前方转弯（前进中左转）**

```
左摇杆 ↑ 幅度较小（如 30%）
右摇杆 ↑ 幅度较大（如 80%）

L:  前进慢速   →  ──►  （弱）
R:  前进快速   →  ══►  （强）

效果：机器人整体前进，向左弧线偏转
      转弯半径由两摇杆幅度差决定，差值越大转弯越急
```

---

**② 向右前方转弯（前进中右转）**

```
左摇杆 ↑ 幅度较大（如 80%）
右摇杆 ↑ 幅度较小（如 30%）

L:  前进快速   →  ══►  （强）
R:  前进慢速   →  ──►  （弱）

效果：机器人整体前进，向右弧线偏转
      转弯半径由两摇杆幅度差决定，差值越大转弯越急
```

---

**③ 向左后方转弯（后退中左转）**

```
左摇杆 ↓ 幅度较大（如 80%）
右摇杆 ↓ 幅度较小（如 30%）

L:  后退快速   →  ◄══  （强）
R:  后退慢速   →  ◄──  （弱）

效果：机器人整体后退，向左弧线偏转
```

---

**④ 向右后方转弯（后退中右转）**

```
左摇杆 ↓ 幅度较小（如 30%）
右摇杆 ↓ 幅度较大（如 80%）

L:  后退慢速   →  ◄──  （弱）
R:  后退快速   →  ◄══  （强）

效果：机器人整体后退，向右弧线偏转
```

---

#### 转弯半径参考

| 左摇杆 Y | 右摇杆 Y | 行为 | 转弯半径 |
|---------|---------|------|---------|
| +1.0 | +1.0 | 直行前进 | 无穷大（直线）|
| +0.3 | +1.0 | 前进左转 | 大弧线（缓转）|
| 0.0 | +1.0 | 前进左转 | 中等弧线（左轮原地，右轮驱动）|
| -0.5 | +1.0 | 原地急转（偏左前）| 极小（类原地）|
| -1.0 | +1.0 | 原地左转 | 车身中心点（最小半径）|
| +1.0 | -1.0 | 原地右转 | 车身中心点（最小半径）|

### 5.3 速度控制

摇杆偏离中心的距离决定电机速度（线性映射）：

```
归一化值 normY ∈ [-1.0, +1.0]
  normY = +1.0  →  PWM duty = 255（全速前进）
  normY =  0.0  →  PWM duty =   0（停止/滑行）
  normY = -1.0  →  PWM duty = 255（全速后退）

实际发送指令（每 20ms = 50Hz 发送一次）：
  {"cmd":"motor","a":<-255~255>,"b":<-255~255>}
```

### 5.4 松手自动停止

松开摇杆后，摇杆弹回中心，摇杆值归零，电机进入**滑行（Coast）** 状态，自然减速停止。

## 6. 激光发射器

### 6.1 开启 / 关闭激光

点击中央控制面板中的 **⚡（闪电图标）** 按钮：

| 按钮状态 | 含义 | 背景色 |
|---------|------|-------|
| 暗色 | 激光关闭 | 深灰色 |
| 亮色 | 激光开启 | 橙色（`#FF9800`）|

每次点击在「开 / 关」之间切换，发送指令：

```json
{"cmd":"laser","on":true}   // 开启
{"cmd":"laser","on":false}  // 关闭
```

### 6.2 激光硬件参数

| 参数 | 值 |
|-----|----|
| 波长 | 980 nm（近红外，人眼不可见）|
| 功率 | 30 mW |
| 载波频率 | **38 kHz** TTL PWM（匹配 VS1838B 带通滤波器）|
| 占空比 | 50%（duty = 128 / 256）|
| 控制引脚 | ESP32-S3 GPIO 11 |

> 激光开启时，ESP32 在 GPIO 11 输出 38 kHz PWM 方波，驱动激光模组发射调制红外光束。VS1838B 接收管只对 38 kHz 调制信号有响应，避免环境光干扰。

## 7. 激光接收器（IR 命中计数）

### 7.1 实时计数显示

主界面中央下方的绿色大数字即为**激光命中次数**：

```
┌──────────────────┐
│  激光命中次数     │
│       42         │  ← Monospace 字体，绿色，实时更新
└──────────────────┘
```

每当对方的激光射中己方的 VS1838B 接收管，ESP32 检测到 GPIO 12（接收管 OUT 引脚）的一次下降沿（FALLING），计数器 +1，并立即通过 WebSocket 推送：

```json
{"ir_count": 42}
```

App 收到后自动刷新界面，无需手动刷新。

### 7.2 重置命中计数

点击中央控制面板中的 **🔄（刷新图标）** 按钮，将命中计数归零：

```json
{"cmd":"ir_reset"}
```

ESP32 收到后将内部计数器置 0，并推送 `{"ir_count":0}` 确认。

### 7.3 主动查询计数

App 内部可调用 `sendGetIr()` 方法主动请求当前计数（适用于初始化场景）：

```json
{"cmd":"get_ir"}
```

ESP32 收到后立即回复当前 `{"ir_count":<N>}`。

## 8. 急停与安全机制

### 8.1 急停按钮（短路制动）

点击中央面板红色 **🛑（停止图标）** 按钮，触发**短路制动（SHORT BRAKE）**：

```json
{"cmd":"brake"}
```

- 双电机立即刹停（TB6612 制动模式，而非惰行）
- 摇杆缓存值同时归零，下一个 50Hz 周期继续发送 0

> 竞赛场合推荐使用急停而非松手停止，制动距离更短。

### 8.2 自动安全保护

以下情况 App 会**自动制动**：

| 触发场景 | 行为 |
|---------|------|
| App 切换到后台（`onStop`）| 调用 `brake()`，发送制动指令 |
| App 销毁（`onDestroy`）| 断开 WebSocket，释放网络绑定 |
| ViewModel 被清理 | 发送制动 + 断开连接 |

ESP32 侧也有超时保护：若超过设定的 `MOTOR_TIMEOUT_MS` 毫秒未收到任何指令（如手机 Wi-Fi 突然断开），ESP32 自动停止双电机。

## 9. 断开连接

点击「**断开**」按钮（红色，LinkOff 图标）：

1. 停止 50Hz 电机指令循环
2. 关闭 WebSocket 连接（发送 Close Frame code=1000）
3. 向系统注销 `NetworkCallback`
4. 解除进程网络绑定（`bindProcessToNetwork(null)`），手机路由恢复正常

断开后连接状态回到灰色「未连接」，按钮恢复为绿色「连接」。

## 10. 通信协议速查

### App → ESP32（发送）

```json
// 电机速度控制（50Hz，每 20ms 一次）
{"cmd":"motor","a":<-255~255>,"b":<-255~255>}

// 激光开关
{"cmd":"laser","on":true}
{"cmd":"laser","on":false}

// 短路制动急停
{"cmd":"brake"}

// 惰行停止
{"cmd":"stop"}

// 清零 IR 计数
{"cmd":"ir_reset"}

// 主动请求当前 IR 计数
{"cmd":"get_ir"}
```

### ESP32 → App（接收/推送）

```json
// 握手确认（WebSocket 连接建立后 ESP32 主动发送）
{"status":"connected","ip":"192.168.4.1","ssid":"Robot-ESP32"}

// 激光命中计数变化时实时推送（也可由 get_ir 触发）
{"ir_count":42}
```

## 11. 常见问题排查

| 现象 | 可能原因 | 解决方法 |
|------|---------|---------|
| 状态长时间停在「正在连接…」黄色 | ESP32 未开机或热点未启动 | 确认机器人已通电，等待 ESP32 Wi-Fi 指示灯常亮 |
| 弹出「找不到 Robot-ESP32 热点」错误 | SSID/密码与固件不符 | 检查 `AP_SSID`/`AP_PASSWORD` 与固件一致（默认 `Robot-ESP32` / `12345678`）|
| 连接成功但摇杆无响应 | 电机驱动板 STBY 引脚未拉高 | 检查 GPIO 10 → TB6612 STBY 接线 |
| IR 计数不增加 | 激光未对准接收管 / 未开启激光 | 先点「⚡」开启激光，再瞄准 VS1838B 接收窗口 |
| App 切到后台后机器人停止 | 正常安全行为 | 保持 App 在前台运行，避免切换 |
| 手机移动数据断网 | App 绑定 ESP32 AP 后无移动数据 | 正常现象；断开连接后自动恢复移动数据路由 |
| 位置权限被拒绝后无法连接 | `WifiNetworkSpecifier` 需要精确位置权限 | 前往系统设置 → 应用权限 → 重新授予位置权限 |
| 模拟器（AVD）连接失败 | AVD 无物理 Wi-Fi 芯片，`TRANSPORT_WIFI` 不可用 | 必须使用真实 Android 10+ 手机 |
| MIUI 弹出「无法上网」提示 | ESP32 AP 无互联网出口，MIUI 检测到后弹窗 | 点击「继续使用」，App 已绑定网络路由，不影响通信 |
| MIUI 切后台后机器人失控 | MIUI 神隐模式/省电策略杀掉进程 | 设置 → 应用管理 → RobotController → 电池 → 无限制 |
