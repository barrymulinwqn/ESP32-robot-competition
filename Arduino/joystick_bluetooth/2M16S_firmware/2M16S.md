# 2M16S 主控板使用手册

> 2 Motor & 16 Servo Drive Board（2M16S）  
> MCU：**ATmega328P @ 16 MHz**（Arduino UNO 完全兼容，出厂已烧录 optiboot Bootloader）

### 接口位置说明

| 接口 | 板卡位置 | 说明 |
|------|----------|------|
| USB-B（CH340） | 左上角 | 烧录固件 + USB 供电 |
| ISP 6-Pin | 右上角 | Bootloader 烧录，日常不需要 |
| 舵机排针 S1–S8 | 上半区第一排 | PCA9685 通道 0–7，3针（GND/VCC/SIG） |
| 舵机排针 S9–S16 | 上半区第二排 | PCA9685 通道 8–15，3针（GND/VCC/SIG） |
| 电机接口 M1 | 中部左侧 | D4/D5 方向 + D6 PWM，H桥直驱 |
| 电机接口 M2 | 中部右侧 | D7/D8 方向 + D9 PWM，H桥直驱 |
| PS2 手柄接口 | 中下部左侧 | 4线：CLK/CS/CMD/DAT，硬连接 D10–D13 |
| 主电源 VIN | 中下部右侧 | 6–12 V DC，供电机及板载稳压 |
| Arduino 排针（数字） | 下边缘上排 | D0–D13，与标准 UNO 引脚兼容 |
| Arduino 排针（模拟） | 下边缘下排 | A0–A5（A4=SDA, A5=SCL），5V/3.3V/GND |

## 二、板卡概览

| 参数 | 规格 |
|------|------|
| 主控芯片 | ATmega328P（DIP-28 或 SMD） |
| 工作电压 | 5 V（逻辑）/ 6–12 V（电机/舵机电源） |
| 电机通道 | 2 路，板载 H 桥直驱（最大 ~2 A/路） |
| 舵机通道 | 16 路，通过 PCA9685 I²C PWM 扩展（地址 0x40） |
| PS2 接口 | 4 线硬连接（CLK / CS / CMD / DAT） |
| 烧录接口 | USB-B（CH340 USB 转串口）或 6-Pin ISP Header |
| 板载 LED | 与 Pin 13（DAT）共用，PS2 通信时轻微闪烁属正常 |

## 三、针脚与接口详解

### 2.1 PS2 手柄接口（4 线，硬连接不可修改）

板卡将 PS2 信号线**硬连接**到以下 Arduino 引脚，无法通过软件重映射。

| 板卡丝印 | Arduino Pin | 方向 | 说明 |
|----------|-------------|------|------|
| CLK | **D10** | OUT | 时钟线（SPI CLK） |
| CS  | **D11** | OUT | 片选（低有效） |
| CMD | **D12** | OUT | 主机→手柄数据 |
| DAT | **D13** | IN  | 手柄→主机数据，**与板载 LED 共用** |
| VCC | 3.3 V / 5 V | —  | 手柄电源（取决于具体版本） |
| GND | GND | — | 公共地 |

> ⚠️ 本项目使用 **YFRobot 2015 蓝牙游戏手柄解码器**，它将蓝牙信号转为标准 PS2 协议，直接插入上述 4 线接口即可。

---

### 2.2 电机驱动接口（M1 / M2，板载 H 桥）

| 功能 | Arduino Pin | 说明 |
|------|-------------|------|
| M1 方向 A | **D4** | 配合 M1B 决定 M1 转向 |
| M1 方向 B | **D5** | 配合 M1A 决定 M1 转向 |
| M1 使能（PWM） | **D6** | Timer0 OC0A，调速 PWM，不影响 millis/delay |
| M2 方向 A | **D7** | 配合 M2B 决定 M2 转向 |
| M2 方向 B | **D8** | 配合 M2A 决定 M2 转向 |
| M2 使能（PWM） | **D9** | Timer1 OC1A，调速 PWM |

**H 桥逻辑真值表：**

| DIR A | DIR B | PWM | 电机状态 |
|-------|-------|-----|----------|
| HIGH  | LOW   | >0  | 正转（前进） |
| LOW   | HIGH  | >0  | 反转（后退） |
| LOW   | LOW   | 0   | 滑行停止（Coast） |
| HIGH  | HIGH  | 0   | 短路制动（Short Brake）⚡ 急停首选 |

> 短路制动：两端同时拉高，电机绕组短路产生反向制动力矩，停车最迅速。

---

### 2.3 舵机接口（S1–S16，PCA9685 扩展）

| 接口 | 说明 |
|------|------|
| S1 – S16 | 3 针排针（GND / VCC / SIG），标准 50 Hz PWM 舵机 |
| I²C 地址 | 0x40（默认，可通过板载焊点更改） |
| 电源 | 独立舵机电源输入（建议 5–6 V，与逻辑电源隔离） |

> 本固件（`2M16S_firmware.ino`）暂未使用 PCA9685，16 路舵机接口留作扩展。

---

### 2.4 UART 串口（与 Digital PWM UNO 通信）

| 板卡引脚 | Arduino Pin | 方向 | 连接目标 | 说明 |
|----------|-------------|------|----------|------|
| TX | **D1** | OUT | UNO D4（SoftwareSerial RX） | 发送手柄状态帧，115200 bps |
| RX | **D0** | IN  | UNO D5（SoftwareSerial TX） | 接收激光命中计数（可选） |

> ⚠️ **烧录固件时必须断开 D0 / D1 的外部连线**，否则 USB 串口与外部设备争用会导致上传失败。

**通信帧格式（二进制）：**

发往 UNO（7 字节）：
```
[0xAA][LY][LX][RX][BTN][CHK][0x55]
```
| 字节 | 含义 |
|------|------|
| 0xAA | 帧头 |
| LY   | 左摇杆 Y（0–255，中位 128） |
| LX   | 左摇杆 X（0–255，中位 128） |
| RX   | 右摇杆 X（0–255，中位 128） |
| BTN  | bit0:L1 / bit1:L2 / bit2:R1 / bit3:R2 |
| CHK  | LY ^ LX ^ RX ^ BTN（异或校验） |
| 0x55 | 帧尾 |

来自 UNO（6 字节，可选回传）：
```
[0xBB][IR3][IR2][IR1][IR0][CHK]
```
IR3–IR0 为 uint32 命中计数（大端字节序），CHK = IR3 ^ IR2 ^ IR1 ^ IR0。

---

### 2.5 电源接口

| 接口 | 说明 |
|------|------|
| VIN / GND | 主电源输入（6–12 V，供电机及板载稳压） |
| 5V / GND  | 5 V 稳压输出（来自板载 LDO 或 USB） |
| USB-B     | USB 供电 + 串口烧录（CH340G） |

---

### 2.6 ISP 烧录接口（6-Pin）

| Pin | 信号 | 说明 |
|-----|------|------|
| 1   | MISO | 主入从出 |
| 2   | VCC  | 5 V |
| 3   | SCK  | 时钟 |
| 4   | MOSI | 主出从入 |
| 5   | RST  | 复位（低有效） |
| 6   | GND  | 地 |

> ISP 接口主要用于直接烧录 Bootloader，日常固件更新推荐使用 USB 方式。

## 四、烧录固件详细步骤

### 3.1 环境准备

1. 安装 **Arduino IDE**（推荐 2.x）：https://www.arduino.cc/en/software  
2. 打开 Arduino IDE，进入 **工具 → 开发板管理器**，确认已安装 `Arduino AVR Boards`（内置，包含 Arduino UNO）。  
3. 安装依赖库——打开 **工具 → 库管理器**，搜索并安装：  
   - **PS2X_lib**（by Bill Porter）  
     > 若库管理器找不到，请从 https://github.com/madsci1016/Arduino-PS2X 下载 `.zip`，通过 **项目 → 加载库 → 添加 .ZIP 库** 安装。
   - **Wire**（内置，无需额外安装）

---

### 3.2 硬件连接（烧录前必做）

> ⚠️ **关键：断开 D0 / D1 外部连线**

在连接 USB 数据线之前，必须将 `Pin 0 (RX)` 和 `Pin 1 (TX)` 与 UNO 之间的杜邦线**全部拔除**，否则：
- 外部设备会在烧录时向串口发送信号，干扰 ATmega328P 接收来自 PC 的固件数据。
- 上传报错：`avrdude: stk500_recv(): programmer is not responding`

1. 拔除 D0 / D1 连线  
2. 用 **USB-B 数据线**将 2M16S 板连接到电脑  
3. 检查电脑设备管理器（Windows）或 `/dev/tty.*`（macOS/Linux）中是否出现 CH340 串口

---

### 3.3 Arduino IDE 配置

| 设置项 | 值 |
|--------|----|
| 开发板 | **Arduino UNO** |
| 处理器 | ATmega328P |
| 端口   | 选择对应的 CH340 串口（如 `/dev/cu.usbserial-XXXX` 或 `COM3`） |
| 波特率（上传器） | 115200（IDE 自动处理，无需手动设置） |

> macOS 若识别不到 CH340，请安装驱动：  
> https://www.wch.cn/download/CH341SER_MAC_ZIP.html

---

### 3.4 打开并上传固件

1. 在 Arduino IDE 中打开：  
   `Arduino/joystick_bluetooth/2M16S_firmware/2M16S_firmware.ino`

2. 点击左上角 **✓（验证/编译）** 按钮，确认无编译错误。

3. 点击 **→（上传）** 按钮，IDE 底部状态栏显示进度：
   ```
   Sketch uses XXXX bytes (XX%) of program storage space.
   Global variables use XXXX bytes (XX%) of dynamic memory.
   ```

4. 看到 `上传成功` / `Done uploading` 即完成烧录。

---

### 3.5 烧录后验证

1. 重新连接 D0 / D1 杜邦线。  
2. 打开 Arduino IDE **串口监视器**（115200 bps），观察调试输出：
   - 正常：每 20 ms 输出手柄状态帧
   - 异常：`PS2 init failed, retrying...`（检查 YFRobot 解码器电源和接线）
3. 推动手柄摇杆，确认电机响应正确：
   - 左摇杆上推 → 两电机正转（前进）
   - 左摇杆左/右推 → 差速转弯
   - 右摇杆左/右推 → 原地旋转
   - L1 按下 → 紧急制动（电机立即停止）

---

### 3.6 常见问题排查

| 现象 | 原因 | 解决方法 |
|------|------|----------|
| 上传失败，`not responding` | D0/D1 未断开 | 拔除 D0/D1 外部连线再上传 |
| 上传失败，找不到串口 | CH340 驱动未安装 | 安装对应系统的 CH340 驱动 |
| 电机不转 | 电机电源未接或引脚映射错误 | 检查 VIN 接线，对照原理图核实 D4–D9 映射 |
| PS2 手柄无响应 | 解码器供电或 CLK/CS/CMD/DAT 接线错误 | 检查 YFRobot 2015 模块接线和电源 |
| 舵机抖动/不受控 | PCA9685 未初始化（本固件未使用） | 如需舵机功能需在代码中加入 PCA9685 初始化 |
