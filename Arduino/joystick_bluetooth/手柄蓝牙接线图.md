## 2. 供电电路

> ⚠️ **D153C 内置降压稳压电路**，不需要额外降压模块。电池电压经 D153C 统一分配给各模块。

```
18650 电池组（2节串联 标称 ~7.4V，满电 ~8.4V）
        │
        ├──[正极]──→ D153C 顶部螺丝端子 Vin+
        │                   │
        │          [ON/OFF 拨动开关]
        │                   │
        │          D153C 内置降压稳压电路
        │            ├── 5V 输出 ──→ ESP32-S3 5V 引脚
        │            ├── 5V 输出 ──→ 激光发射器 VCC
        │            ├── 5V 输出 ──→ Digital PWM UNO 5V
        │            ├── 5V 输出 ──→ 2 Motor && 16 Servo Board VCC
        │            ├── 3V3 输出 ──→ VS1838B VCC
        │            └── Vout+ ──→ 备用（电池电压直通）
        │
        └──[负极]──→ D153C 顶部螺丝端子 -（共地，所有模块共同接地）
```

> **重要**：所有模块 GND 必须短接至同一 GND 汇流点（D153C 顶部端子 "-"）

## 3. 游戏手柄解码器（YFRobot 2015）→ 2 Motor && 16 Servo Drive Board 接线

### 3.1 YFRobot 2015 解码器引脚说明

YFRobot 2015 是蓝牙游戏手柄的硬件解码模块，内部将蓝牙手柄信号转换为 **PS2 协议**（CLK/CS/CMD/DAT 四线 SPI-like 接口），供主控板读取。

| 解码器引脚 | 描述 | 连接至 2M16S 板 |
|:---------:|------|:--------------:|
| **GND** | 电源负极 | GND |
| **VCC** | 电源正极（5V） | 5V |
| **CLK** | PS2 时钟信号（Clock） | **Pin 10** |
| **CS** | PS2 片选信号（Chip Select / ATT） | **Pin 11** |
| **CMD** | PS2 命令信号（MOSI，主→手柄） | **Pin 12** |
| **DAT** | PS2 数据信号（MISO，手柄→主） | **Pin 13** |

> **协议说明**：PS2 手柄协议为同步串行接口，CLK 由主控发出，DAT/CMD 在每个时钟边沿采样。2M16S 板上运行的 Arduino 固件通过 PS2 库（`PS2X_lib`）读取摇杆轴值（0~255）和所有按键状态。

### 3.2 2 Motor && 16 Servo Drive Board 功能说明

| 功能 | 描述 |
|:----:|------|
| 手柄读取 | 通过 Pin10~13（PS2 协议）读取解码器输出的摇杆/按键数据 |
| 指令生成 | 将摇杆轴值换算为差速控制量，格式化为 UART 指令帧 |
| 串口输出 | 通过 TX 引脚向 Digital PWM UNO 发送运动指令 |
| 舵机通道 | S1~S16（16路 50Hz PWM，可选扩展使用） |

### 3.3 2M16S 板引脚对外连接汇总

| 2M16S 引脚 | 描述 | 连接至 |
|:----------:|------|:------:|
| **VCC** | 电源正（5V） | D153C 5V 输出 |
| **GND** | 电源负 | 系统公共 GND |
| **Pin 10** | PS2 CLK 输出 | 解码器 **CLK** |
| **Pin 11** | PS2 CS 输出 | 解码器 **CS** |
| **Pin 12** | PS2 CMD 输出 | 解码器 **CMD** |
| **Pin 13** | PS2 DAT 输入 | 解码器 **DAT** |
| **TX** | 串口发送（手柄状态帧） | Digital PWM UNO **RX**（或 SoftwareSerial 引脚） |
| **S1~S16** | 16路舵机 PWM 输出 | 舵机（本项目暂未使用，可扩展） |

## 4. Digital PWM UNO → ESP32-S3 N8R2 接线

### 4.1 功能说明

Digital PWM UNO（Arduino UNO 兼容板）承担**信号中转与指令格式化**：
- D2（SoftwareSerial RX）接收 2M16S 板手柄原始数据
- 解析摇杆轴值，计算差速控制量（左/右电机速度）
- 通过硬件 UART（D1/TX）向 ESP32 发送 JSON 运动指令

### 4.2 UNO → ESP32 电平转换（必要！）

> ⚠️ UNO D1/TX 输出为 **5V TTL**，而 ESP32-S3 GPIO 最大承受 **3.3V**，必须进行电平转换！

**方案一（推荐）：电阻分压**

```
UNO D1/TX (5V) ─┬─ 10 kΩ ─┬─ ESP32-S3 GPIO18 (3.3V RX)
                │          │
               GND       20 kΩ
                            │
                           GND

输出电压：5V × 20/(10+20) = 3.33V ✓
```

**方案二：双向逻辑电平转换模块（TXS0108E / SN74AHCT125）**

接 HV（5V）、LV（3.3V）、A侧接 UNO TX、B侧接 ESP32 GPIO18，更可靠。

### 4.3 接线对照表

| UNO 引脚 | 方向 | 连接至 | 说明 |
|:--------:|:----:|:------:|------|
| **5V** | ← | D153C 5V 输出 | UNO 供电 |
| **GND** | — | 系统公共 GND | 共地 |
| **D2**（SoftwareSerial RX） | ← | 2M16S 板 **TX** | 接收手柄原始数据 |
| **D3**（SoftwareSerial TX） | → | 2M16S 板 **RX** | 向板子回传（可选） |
| **D1/TX**（硬件 UART TX） | → | [分压] → ESP32-S3 **GPIO18** | 发送运动指令给 ESP32 |
| **D0/RX**（硬件 UART RX） | ← | ESP32-S3 **GPIO17** | 接收 ESP32 回传状态（可选） |

> **ESP32 GPIO17 TX → UNO D0/RX（5V）**：ESP32 输出 3.3V HIGH，Arduino UNO VIH 最低 2.0V，可直连，无需电平转换。

## 9. 运动控制逻辑

### 9.1 游戏手柄按键 → 动作映射

| 手柄操作 | 动作 | 左摇杆 Y | 左摇杆 X | 右摇杆 X |
|:-------:|:----:|:--------:|:--------:|:--------:|
| 左摇杆 **上推** | 前进 | + | 0 | 0 |
| 左摇杆 **下推** | 后退 | − | 0 | 0 |
| 左摇杆 **左上** | 左前（左弧线前进） | + | − | 0 |
| 左摇杆 **右上** | 右前（右弧线前进） | + | + | 0 |
| 左摇杆 **左下** | 左后（左弧线后退） | − | − | 0 |
| 左摇杆 **右下** | 右后（右弧线后退） | − | + | 0 |
| 右摇杆 **左推** | 原地左转 | 0 | 0 | − |
| 右摇杆 **右推** | 原地右转 | 0 | 0 | + |
| 双摇杆 **归中** | 滑行停止（惰行） | 0 | 0 | 0 |
| 按下 **L1 键** | 制动停止（急停） | — | — | — |

### 9.2 TB6612 电机控制真值表

> **定义**：电机 A = 左电机，电机 B = 右电机；PWM 范围 0~255；speed = 手柄摇杆值换算后的占空比

| 动作 | 左电机 (A) | AIN1 | AIN2 | PWMA | 右电机 (B) | BIN1 | BIN2 | PWMB |
|:----:|:----------:|:----:|:----:|:----:|:----------:|:----:|:----:|:----:|
| **前进** | 正转 | HIGH | LOW | speed | 正转 | HIGH | LOW | speed |
| **后退** | 反转 | LOW | HIGH | speed | 反转 | LOW | HIGH | speed |
| **左前**（左弧线）| 慢正/停 | HIGH | LOW | speed×k | 正转 | HIGH | LOW | speed |
| **右前**（右弧线）| 正转 | HIGH | LOW | speed | 慢正/停 | HIGH | LOW | speed×k |
| **左后**（左弧线）| 慢反/停 | LOW | HIGH | speed×k | 反转 | LOW | HIGH | speed |
| **右后**（右弧线）| 反转 | LOW | HIGH | speed | 慢反/停 | LOW | HIGH | speed×k |
| **原地左转** | 反转 | LOW | HIGH | 200 | 正转 | HIGH | LOW | 200 |
| **原地右转** | 正转 | HIGH | LOW | 200 | 反转 | LOW | HIGH | 200 |
| **制动停止** | 制动 | HIGH | HIGH | 0 | 制动 | HIGH | HIGH | 0 |
| **滑行停止** | 惰行 | LOW | LOW | 0 | 惰行 | LOW | LOW | 0 |

> **差速系数 k**：左前/右前时，内侧电机速度 = `speed × (1 − |X轴值|/100)`，实现平滑弧线转向。
> **STBY（GPIO 10）**：必须保持 HIGH，任何状态下置 LOW 会立即切断双路电机输出。

## 10. UART 串口通信协议（Digital PWM UNO → ESP32）

### 10.1 物理层参数

| 参数 | 值 |
|:----:|:--:|
| 波特率 | 115200 bps |
| 数据位 | 8 bit |
| 停止位 | 1 bit |
| 校验 | 无 |
| 电平 | UNO 5V TTL → 分压 → ESP32 3.3V |
| ESP32 UART | UART2（GPIO17 TX / GPIO18 RX） |

### 10.2 指令帧格式（UNO → ESP32）

```json
{"cmd":"move","lspd":200,"rspd":200}     // 前进（全速）
{"cmd":"move","lspd":-200,"rspd":-200}   // 后退
{"cmd":"move","lspd":80,"rspd":200}      // 左前（差速弧线）
{"cmd":"move","lspd":200,"rspd":80}      // 右前（差速弧线）
{"cmd":"move","lspd":80,"rspd":-200}     // 左后（差速弧线）
{"cmd":"move","lspd":-200,"rspd":80}     // 右后（差速弧线）
{"cmd":"move","lspd":-150,"rspd":150}    // 原地左转
{"cmd":"move","lspd":150,"rspd":-150}    // 原地右转
{"cmd":"brake"}                           // 制动停止（L1 键）
{"cmd":"stop"}                            // 滑行停止（摇杆归中）
```

> `lspd` / `rspd`：−255 ~ +255，正值=正转（前进方向），负值=反转（后退方向），0=停止。
> 每帧以 `\n`（换行符）结尾，ESP32 按行读取并解析 JSON。

### 10.3 状态回传帧（ESP32 → UNO，可选）

```json
{"ir_count":12,"battery_mv":7350}
```

```
 │  VCC ← D153C 5V                       │
 │  GND ── 公共地                         │
 └───────────────────────────────────────┘
         ↑ PS2 协议（CLK/CS/CMD/DAT）
 ┌───────────────────────────────────────┐
 │  YFRobot 2015 游戏手柄解码器          │
 │                                       │
 │  ← [BT] ← 蓝牙游戏手柄               │
 │  CLK → 2M16S Pin10                    │
 │  CS  → 2M16S Pin11                    │
 │  CMD → 2M16S Pin12                    │
 │  DAT → 2M16S Pin13                    │
 │  VCC ← D153C 5V                       │
 │  GND ── 公共地                         │
 └───────────────────────────────────────┘

 ┌──────────────┐     ┌──────────────────────┐
 │  激光发射器  │     │   VS1838B 激光接收器  │
 │  VCC→5V(D153C)│     │  VCC → 3V3（D153C）  │
 │  GND→公共地  │     │  GND → 公共地         │
 │  TTL←GPIO11  │     │  OUT → GPIO12         │
 └──────────────┘     └──────────────────────┘
```

## 12. 具体接线步骤（Step-by-Step）

### 第一步：搭建供电电路

1. 将两节 18650 电池**串联**（第一节负极接第二节正极），总电压约 7.4V。
2. 取两根电线（红/黑），接入 D153C 顶部螺丝端子：
   - 红线 → **Vin+**（电池串联正极端）
   - 黑线 → **-**（电池串联负极端）
3. 确保 D153C 拨动开关处于 **OFF** 状态（暂不通电）。

### 第二步：D153C → ESP32-S3 供电与共地

4. 用杜邦线从 **D153C 底部排针 5V** 连接至 **ESP32-S3 5V** 引脚。
5. 用杜邦线从 **D153C 底部排针 GND**（任意一个）连接至 **ESP32-S3 GND**。

### 第三步：ESP32-S3 → TB6612 D153C 控制信号（7根线）

6. `ESP32 GPIO4` → `D153C PWMA`（左电机 PWM 速度）
7. `ESP32 GPIO5` → `D153C AIN1`（左电机方向1）
8. `ESP32 GPIO6` → `D153C AIN2`（左电机方向2）
9. `ESP32 GPIO7` → `D153C PWMB`（右电机 PWM 速度）
10. `ESP32 GPIO8` → `D153C BIN1`（右电机方向1）
11. `ESP32 GPIO9` → `D153C BIN2`（右电机方向2）
12. `ESP32 GPIO10` → `D153C STBY`（驱动使能，HIGH=工作）

### 第四步：TB6612 D153C → 电机（4根线）

13. `D153C AO1` → **左电机正极线（红）**
14. `D153C AO2` → **左电机负极线（黑）**
15. `D153C BO1` → **右电机正极线（红）**
16. `D153C BO2` → **右电机负极线（黑）**

> 如调试时发现电机转向相反，交换该电机的两根接线即可（无需改代码）。

### 第五步：激光发射器接线（3根线）

17. 激光模组 **VCC** → `D153C 5V 输出`
18. 激光模组 **GND** → 公共 GND
19. 激光模组 **TTL/Signal** → `ESP32 GPIO11`

### 第六步：VS1838B 激光接收器接线（3根线）

> VS1838B 正面（凸面朝向自己），引脚从左到右：OUT、GND、VCC。

20. VS1838B **Pin3（VCC）** → `D153C 3V3 输出`
21. VS1838B **Pin2（GND）** → 公共 GND
22. VS1838B **Pin1（OUT）** → `ESP32 GPIO12`

### 第七步：D153C → Digital PWM UNO 供电

23. `D153C 5V` → **UNO 5V 引脚**（或 UNO VIN 接 D153C Vout+ 均可）
24. 公共 GND → **UNO GND 引脚**

### 第八步：YFRobot 2015 解码器接线（6根线）

25. 解码器 **GND** → 公共 GND
26. 解码器 **VCC** → `D153C 5V 输出`
27. 解码器 **CLK** → **2M16S 板 Pin 10**
28. 解码器 **CS** → **2M16S 板 Pin 11**
29. 解码器 **CMD** → **2M16S 板 Pin 12**
30. 解码器 **DAT** → **2M16S 板 Pin 13**

> PS2 协议 CLK/CS/CMD 均由 2M16S 主控输出，DAT 由解码器（手柄端）输出至 2M16S 输入。引脚均为 5V TTL，直接连接即可。

### 第九步：2 Motor && 16 Servo Drive Board 供电与串口

31. `D153C 5V` → **2M16S VCC**
32. 公共 GND → **2M16S GND**
33. `2M16S TX`（手柄状态帧输出）→ **UNO RX 引脚**（硬件 UART 或 SoftwareSerial 接收引脚）

> 若 2M16S TX 为 5V TTL 接 UNO 5V RX，可直连；若接 ESP32 GPIO 须经电平转换。

### 第十步：Digital PWM UNO → ESP32-S3 接线（含电平转换）

**电平转换电路（在 UNO D1/TX 与 ESP32 GPIO18 之间）：**

```
UNO D1/TX ──┬── 10 kΩ ──┬── ESP32-S3 GPIO18
             │            │
            GND        20 kΩ
                           │
                          GND
```

34. 按上图焊接 10kΩ 和 20kΩ 电阻（可在面包板上搭建）。
35. `UNO D1/TX` → 分压电路输入端 → `ESP32 GPIO18`（UART2 RX）
36. `ESP32 GPIO17`（UART2 TX） → `UNO D0/RX`（3.3V 电平可直连 5V Arduino 输入端）

### 第十一步：通电测试顺序

37. 确认所有接线无误，无短路。
38. 将 D153C 拨动开关拨至 **ON**。
39. ESP32-S3、UNO 和 2M16S 板均上电；解码器随 2M16S 板一起上电。
40. 蓝牙手柄开机（通常长按 HOME/PS 键），等待与**解码器**自动配对（手柄指示灯常亮表示配对成功）。
41. 配对成功后 2M16S 板开始输出手柄状态帧，推动左摇杆，观察双电机响应；归中后电机应停止运转。
42. 按下 **L1 键**，验证制动停止（电机立即锁止）。

## 13. 注意事项与安全提醒

| 序号 | 事项 | 说明 |
|:----:|------|------|
| 1 | **必须共地** | 电池、D153C、ESP32、UNO、2M16S 板、激光模组所有 GND 必须连通，否则 UART 信号异常 |
| 2 | **5V→3.3V 电平转换** | UNO D1/TX 直接接 ESP32 GPIO 会导致 GPIO 过压损坏，必须经分压或电平模块转换 |
| 3 | **禁用 GPIO** | ESP32-S3 N8R2 的 GPIO26~32（Flash）、GPIO35~37（PSRAM）禁止用作普通 IO |
| 4 | **STBY 常高** | 不需要动态控制休眠时可将 STBY 直接接 3.3V，释放 GPIO10；代码控制时保持 HIGH |
| 5 | **电机转向校正** | 若某路电机转向反向，交换该路电机两根输出线（AO1↔AO2 或 BO1↔BO2），不要修改代码 |
| 6 | **激光安全** | 980nm 30mW 近红外激光**不可见但有害**，严禁对准眼睛直视 |
| 7 | **手柄配对** | 2M16S 板上电后进入配对模式，手柄开机后按 HOME 键开始配对；已配对设备断电后重新上电可自动重连 |
| 8 | **电池放电** | 两个 12V 电机在 7.4V 下启动电流可达 2~4A，确保 18650 持续放电倍率 ≥ 2C（推荐 25R/30Q 型） |
| 9 | **超时保护** | ESP32 固件应设置 UART 超时（建议 500ms），若超过时间未收到指令则自动执行滑行停止 |
| 10 | **串口占用冲突** | UNO D0/RX、D1/TX 烧录固件时需断开与 ESP32 的连线，烧录完成后再重新接上 |
